# Транспортна мережа міста: граф + DFS/BFS + Дейкстра

## Опис
У цьому домашньому завданні я вирішив об’єднати всі три підзавдання в один цілісний код.
Реалізацію пошуку шляхів я виконав вручну, щоб краще спробувати зрозуміти роботу алгоритмів (DFS, BFS, Дейкстра).

Створено простий граф міста з 6 районів (A,B,C,D,E,F). Ребра — це дороги, вага = час у хвилинах.
Використав `networkx` для побудови і візуалізації.

## Завдання 1
- Створено граф, намальовано схему.
- Пораховано: кількість вершин/ребер, ступені вершин.

## Завдання 2
- Реалізовано`DFS` і `BFS` для пошуку шляху (наприклад A -> F).
- Порівняння: `BFS` дає шлях з мінімальною кількістю ребер, `DFS` просто перший знайдений у глибину (може бути довший).
- Пояснення: BFS шукає шар за шаром, тому мінімізує к-сть кроків; DFS більше занурюється вглиб.

## Завдання 3
- Додав ваги до ребер (час).
- Реалізовано алгоритм Дейкстри (без heap, просто).
- Знайшов найкоротші шляхи за вагою між усіма вершинами (запуск Дейкстри від кожної вершини).
- Вивів приклади шляхів і повну таблицю мінімальних часів.
